# Data — share-of-citation tracking

This folder holds the schema and templates for logging audit results over time. The CSV templates are deliberately simple — one row per observation, one file per audit pass — so that they remain portable across spreadsheet tools, BI platforms, and analysis notebooks.

## Files

- `share_of_citation_template.csv` — schema for the **response audit** (see `../prompts/`). One row per (date, query, model, brand) tuple.
- `probe_output.csv` *(generated by `../probes/embedding_probe.py`)* — schema for the **embedding probe**. One row per (date, model, brand, attribute) tuple.

## Workflow

1. Copy `share_of_citation_template.csv` and rename it with the audit date — for example, `audit_2026_05.csv`.
2. Run the audit (manually or via the API pipeline). Fill the rows.
3. Have a second coder review a 20% sample. Reconcile.
4. Append the file to a longitudinal store (database, lakehouse, or a simple `audits/` folder versioned in git).
5. Aggregate for reporting:
   - **Share-of-citation** = mean(`citation_provided`) by (model, month).
   - **Mention rate** = mean(`brand_mentioned`) by (model, month).
   - **Position-weighted citation index** = mean(weighted `position`) by (model, month), with weights position 1 = 3, position 2 = 2, position 3 = 1, position 0 = 0.
   - **Sentiment ratio** = (positive − negative) / total mentions.
   - **Hallucination rate** = mean(`hallucination`) by (model, month).

## A note on confidentiality

If you run audits on competitive brands, the resulting data is competitively sensitive. The repository ships with empty templates, never real audit data. Real audits are stored under access-controlled infrastructure outside this repository.

## Integration with the embedding probe

The two instruments measure different things and should be reported together. A divergence — for instance, high embedding proximity but low share-of-citation — is itself a finding and usually points at one of three causes:

1. The brand has a weak presence in the retrieval index (corpus presence problem).
2. The brand has a weak entity-graph anchor (named-entity recognition problem).
3. The brand's content is paraphrase-friendly and gets reused without attribution (information gain problem).

Each cause has a different remedy. The triangulation is what makes the audit actionable.
